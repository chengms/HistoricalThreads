# GitHub Pages 快速部署步骤

## 当前状态
✅ 代码已推送到 GitHub  
✅ GitHub Actions 工作流已配置  
✅ Vite base 路径已设置  

## 立即操作步骤

### 1. 检查 GitHub Actions 状态

1. 访问：https://github.com/chengms/HistoricalThreads/actions
2. 查看是否有 "Deploy to GitHub Pages" 工作流
3. 如果看到工作流：
   - ✅ **绿色勾号** = 部署成功，等待几分钟后访问网站
   - ⏳ **黄色圆圈** = 正在部署，请等待
   - ❌ **红色叉号** = 部署失败，点击查看错误日志

### 2. 如果工作流未运行，手动触发

1. 在 Actions 页面，点击左侧的 "Deploy to GitHub Pages"
2. 点击右侧的 **"Run workflow"** 按钮
3. 选择 `main` 分支
4. 点击 **"Run workflow"** 确认

### 3. 等待部署完成

- 部署通常需要 2-5 分钟
- 可以在 Actions 页面实时查看进度
- 看到绿色勾号表示部署成功

### 4. 访问网站

部署成功后，访问：
```
https://chengms.github.io/HistoricalThreads/
```

## 常见问题

### Q: 看到 404 错误？
A: 等待 2-3 分钟后刷新，GitHub Pages 需要时间生效

### Q: 工作流运行失败？
A: 检查 Actions 日志，常见原因：
- Node.js 版本不兼容
- 依赖安装失败
- 构建错误

### Q: 如何更新网站？
A: 只需推送代码到 main 分支，会自动触发部署
