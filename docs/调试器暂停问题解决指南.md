# 调试器暂停问题解决指南

## 问题描述

刷新网页时，浏览器开发者工具总是停在某个断点处，导致页面无法正常加载。

## 可能原因

1. **浏览器开发者工具中设置了断点**
2. **浏览器设置了"在异常时暂停"**
3. **浏览器扩展干扰**
4. **Source Map 配置问题**

## 解决方法

### 方法 1: 清除所有断点（推荐）

1. 打开浏览器开发者工具（F12）
2. 切换到 **Sources** 标签页
3. 在右侧面板找到 **Breakpoints** 区域
4. 点击 **"Deactivate breakpoints"**（停用所有断点）按钮
   - 或者右键点击断点列表，选择 **"Remove all breakpoints"**（移除所有断点）
5. 刷新页面

### 方法 2: 关闭异常暂停

1. 打开浏览器开发者工具（F12）
2. 切换到 **Sources** 标签页
3. 在右侧面板找到 **Breakpoints** 区域
4. 取消勾选以下选项：
   - ❌ **"Pause on uncaught exceptions"**（在未捕获的异常时暂停）
   - ❌ **"Pause on caught exceptions"**（在捕获的异常时暂停）
5. 刷新页面

### 方法 3: 清除浏览器断点数据

1. 打开浏览器开发者工具（F12）
2. 按 `Ctrl+Shift+P`（Windows）或 `Cmd+Shift+P`（Mac）打开命令面板
3. 输入 `Clear breakpoints` 并执行
4. 刷新页面

### 方法 4: 禁用浏览器扩展

某些浏览器扩展可能会干扰调试：

1. 打开浏览器设置
2. 进入扩展/插件管理
3. 临时禁用所有扩展
4. 刷新页面测试
5. 如果问题解决，逐个启用扩展找出问题扩展

### 方法 5: 使用无痕模式

1. 打开浏览器无痕/隐私模式（`Ctrl+Shift+N` 或 `Cmd+Shift+N`）
2. 访问网站
3. 如果无痕模式下正常，说明是浏览器设置或扩展问题

### 方法 6: 清除浏览器缓存和存储

1. 打开浏览器设置
2. 清除浏览数据
3. 选择清除：
   - 缓存
   - Cookie 和网站数据
   - 本地存储
4. 刷新页面

## 预防措施

### 开发时建议

1. **使用条件断点**：只在特定条件下暂停
2. **及时清除断点**：调试完成后移除断点
3. **使用 `console.log` 代替断点**：对于简单调试，使用日志输出

### 代码检查

确保代码中没有 `debugger` 语句：

```bash
# 在项目根目录运行
grep -r "debugger" frontend/src
```

如果找到 `debugger` 语句，应该移除它们。

## 快速检查清单

- [ ] 清除所有断点
- [ ] 关闭异常暂停选项
- [ ] 检查是否有浏览器扩展干扰
- [ ] 尝试无痕模式
- [ ] 清除浏览器缓存
- [ ] 检查代码中是否有 `debugger` 语句

## 如果问题仍然存在

如果以上方法都无法解决问题，请检查：

1. **浏览器版本**：更新到最新版本
2. **开发者工具版本**：确保使用最新版本
3. **项目配置**：检查 `vite.config.ts` 中的 source map 配置
4. **网络问题**：检查是否有网络请求导致的问题

## 相关文件

- `frontend/vite.config.ts` - Vite 构建配置
- `frontend/src/**/*.tsx` - 源代码文件
