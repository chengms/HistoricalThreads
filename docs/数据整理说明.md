## 数据格式说明

### 朝代数据格式
```json
{
  "id": 1,
  "name": "朝代名称",
  "startYear": 起始年份（负数表示公元前）,
  "endYear": 结束年份,
  "description": "朝代简介"
}
```

### 事件数据格式
```json
{
  "id": 1,
  "title": "事件标题",
  "description": "事件详细描述",
  "eventYear": 事件年份,
  "eventType": "事件类型（political/economic/cultural/military/reform/other）",
  "dynastyId": 所属朝代ID,
  "location": "发生地点",
  "persons": [相关人物ID数组],
  "sources": [来源ID数组]
}
```

### 人物数据格式
```json
{
  "id": 1,
  "name": "人物姓名",
  "nameVariants": ["别名数组"],
  "birthYear": 出生年份,
  "deathYear": 死亡年份,
  "biography": "人物简介",
  "personType": ["人物类型数组（politician/economist/writer/artist/philosopher/scientist/military/other）"],
  "dynastyId": 所属朝代ID,
  "birthplace": "出生地",
  "sources": [来源ID数组]
}
```

### 关系数据格式
```json
{
  "id": 1,
  "fromPersonId": 起始人物ID,
  "toPersonId": 目标人物ID,
  "relationshipType": "关系类型（teacher_student/colleague/enemy/family/friend/mentor/influence/cooperation/other）",
  "description": "关系描述",
  "startYear": 关系开始年份（可选）,
  "endYear": 关系结束年份（可选）,
  "strength": 关系强度（1-5）
}
```

### 来源数据格式
```json
{
  "id": 1,
  "title": "来源标题",
  "author": "作者（可选）",
  "sourceType": "来源类型（academic_book/official_history/museum/authoritative_website/research_paper/archive）",
  "publisher": "出版社（可选）",
  "publishDate": "出版日期（可选）",
  "credibilityLevel": 可信度等级（1-5）,
  "verified": 是否已验证,
  "url": "来源链接（可选）"
}
```

## 数据更新说明

### 如何添加新数据

1. **添加新朝代**
   - 编辑 `frontend/public/data/dynasties.json`
   - 确保 ID 唯一且递增
   - 填写完整的朝代信息

2. **添加新事件**
   - 编辑 `frontend/public/data/events.json`
   - 关联相关人物和来源
   - 确保事件类型正确

3. **添加新人物**
   - 编辑 `frontend/public/data/persons.json`
   - 填写人物详细信息
   - 关联所属朝代和来源

4. **添加新关系**
   - 编辑 `frontend/public/data/relationships.json`
   - 确保人物ID存在
   - 填写关系类型和描述

5. **添加新来源**
   - 编辑 `frontend/public/data/sources.json`
   - 确保来源权威可靠
   - 填写完整的来源信息

### 数据验证

提交新数据前，请确保：
- ✅ 所有ID唯一且递增
- ✅ 所有关联ID（dynastyId、persons、sources等）都存在
- ✅ 年份格式正确（负数表示公元前）
- ✅ 必填字段完整
- ✅ 数据格式符合JSON规范
