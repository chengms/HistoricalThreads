# 中国历史时间线网站 - 项目需求文档

## 一、项目概述

### 1.1 项目目标
创建一个交互式的中国历史时间线网站，通过时间线和关系图谱可视化展示中国历史事件、人物及其相互关系，提供权威的历史信息查询平台。

### 1.2 核心功能
- **时间线展示**：按时间顺序展示历史事件、人物活动
- **关系图谱**：可视化展示人物之间的关系网络
- **多维分类**：按朝代、事件类型、人物类型（政治/经济/文化）分类
- **信息来源标注**：每条信息标注官方权威来源
- **用户建议**：允许用户提交内容建议和修正
- **搜索功能**：支持关键词搜索历史事件和人物

## 二、功能需求

### 2.1 时间线视图
- **时间轴展示**
  - 支持年/月/日多级时间粒度
  - 可缩放时间范围（从远古到现代）
  - 时间轴可拖拽、缩放
  - 支持按朝代筛选

- **事件展示**
  - 事件卡片：标题、时间、描述、相关人物、来源链接
  - 事件分类：政治事件、经济事件、文化事件、战争、改革等
  - 事件关联：显示相关的前后事件

- **人物展示**
  - 人物卡片：姓名、生卒年、身份、简介、头像
  - 人物分类：政治家、经济学家、文学家、艺术家等
  - 人物时间线：展示人物一生的重要节点

### 2.2 关系图谱视图
- **节点类型**
  - 人物节点：不同颜色区分人物类型
  - 事件节点：显示关键历史事件
  - 朝代节点：展示朝代更迭

- **关系类型**
  - 人物关系：师生、同僚、敌对、家族、朋友等
  - 事件关系：因果关系、时间顺序、影响关系
  - 人物-事件关系：参与、主导、影响

- **交互功能**
  - 节点点击查看详情
  - 节点拖拽调整布局
  - 关系线悬停显示关系说明
  - 支持筛选特定关系类型

### 2.3 信息来源管理
- **来源要求**
  - 每条信息必须标注至少一个官方来源
  - 来源类型：学术著作、官方史料、博物馆、权威网站
  - 来源链接：可点击跳转到原始资料

- **来源展示**
  - 在详情页面显示完整来源列表
  - 支持来源验证状态（已验证/待验证）
  - 显示来源可信度评级

### 2.4 用户建议功能
- **建议类型**
  - 新增内容建议：事件、人物、关系
  - 内容修正建议：错误信息更正
  - 来源补充建议：添加更多权威来源

- **建议流程**
  - 用户提交建议表单
  - 管理员审核建议
  - 审核通过后更新内容
  - 建议状态跟踪（待审核/已通过/已拒绝）

### 2.5 搜索与筛选
- **搜索功能**
  - 全文搜索：事件、人物、朝代
  - 高级搜索：按时间范围、类型、朝代筛选
  - 搜索结果高亮显示

- **筛选功能**
  - 按朝代筛选
  - 按事件类型筛选
  - 按人物类型筛选
  - 按时间范围筛选

## 三、非功能需求

### 3.1 性能要求
- 页面加载时间 < 3秒
- 时间线滚动流畅（60fps）
- 关系图谱渲染流畅（支持1000+节点）
- 支持大数据量（10万+历史记录）

### 3.2 用户体验
- 界面美观、现代化设计
- 动画过渡流畅自然
- 响应式设计，支持多设备
- 操作直观，学习成本低

### 3.3 数据质量
- 信息来源权威可靠
- 数据准确性验证机制
- 定期更新和维护

## 四、技术约束

### 4.1 前端技术
- React 18+ / Vue 3+（推荐React）
- TypeScript
- 时间线可视化库：vis-timeline / Timeline.js
- 关系图谱库：D3.js / vis-network / Cytoscape.js
- UI框架：Ant Design / Material-UI

### 4.2 后端技术
- Node.js / Python（推荐Node.js + Express）
- 数据库：PostgreSQL / MySQL
- 全文搜索：Elasticsearch / PostgreSQL全文搜索

### 4.3 部署要求
- 支持云部署（Vercel / Netlify / 阿里云）
- CDN加速静态资源
- 数据库备份机制

## 五、数据模型

### 5.1 核心实体
- **朝代（Dynasty）**：名称、起止时间、简介
- **事件（Event）**：标题、时间、描述、类型、朝代、来源
- **人物（Person）**：姓名、生卒年、身份、简介、朝代、来源
- **关系（Relationship）**：关系类型、起始节点、结束节点、描述
- **来源（Source）**：标题、URL、类型、可信度、验证状态
- **建议（Suggestion）**：类型、内容、提交人、状态、审核意见

### 5.2 数据关系
- 朝代 1:N 事件
- 朝代 1:N 人物
- 事件 N:M 人物（参与关系）
- 人物 N:M 人物（关系网络）
- 事件 1:N 来源
- 人物 1:N 来源

## 七、数据来源参考

### 7.1 官方权威来源
- 中国历史研究院
- 国家图书馆古籍资源
- 各大博物馆官方网站
- 权威学术出版社（中华书局、商务印书馆等）
- 高校历史系研究成果

### 7.2 数据格式要求
- 所有时间使用统一格式（公元纪年）
- 人物生卒年需标注朝代
- 事件需标注具体时间或时间范围
- 来源链接需可访问且稳定
